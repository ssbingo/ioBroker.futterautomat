## Адаптер futterautomat для ioBroker

Управление (переоборудованными) автоматическими кормушками. Адаптер по расписанию включает и
выключает до **5 свободно выбираемых переключателей** (существующих объектов ioBroker) на
настраиваемое время («кормление»). При необходимости учитываются температура воздуха и воды, а
по геокоординатам вычисляется положение солнца, чтобы не кормить ночью.

### Возможности

* До 5 переключателей, каждый со свободно задаваемым именем (= отдельная вкладка настроек).
* Два режима кормления для каждого переключателя:
  * **Фиксированное время** (например, 08:00 и 18:00).
  * **Интервал в пределах временного окна** (например, каждые 60 мин с 08:00 до 18:00).
* Настраиваемая **длительность кормления в секундах** для каждого переключателя.
* **Блокировка по температуре**: блокировать кормление ниже или выше свободно задаваемых
  значений температуры воды и/или воздуха.
* **Защита от кормления ночью** по восходу/закату с настраиваемым смещением (утро/вечер).
* **Ручной запуск** (`feedNow`) для каждого переключателя и **кнопка «Покормить сейчас»** с
  выбираемой длительностью; при желании в обход всех блокировок.
* **Контроль переключения** для каждого переключателя: проверяет, действительно ли
  переключатель включился и выключился (нужен переключатель, сообщающий своё состояние,
  `ack=true`), с опциональными **уведомлениями Telegram** (успешное кормление / не удалось /
  неисправность выключения).

### Настройка

**Вкладка «Основные настройки»**
* **Местоположение (обязательно)**: использовать системные настройки *или* задать вручную —
  через поиск адреса и метку на карте OpenStreetMap (ключ API не нужен).
* **Солнечное окно**: смещения в минутах после восхода / до заката.
* **Источники температуры**: включить температуру воздуха и воды и выбрать соответствующий
  объект.
* **Переключатели**: добавить переключатели (макс. 5), задать имена, выбрать объект,
  активировать.

**Вкладка переключателя** (создаётся динамически, с именем переключателя)
* Режим (фиксированное время / интервал), время или окно + интервал, длительность кормления,
  блокировка по температуре, ночные/ручные опции, контроль переключения, уведомления Telegram,
  кнопка ручного кормления.

### Точки данных

Глобально:
* `info.connection` – адаптер работает / конфигурация корректна
* `airTemperature`, `waterTemperature` – текущие измеренные температуры
* `sunrise`, `sunset` – рассчитанное время

Для каждого переключателя в `switches.<id>.`:
* `feedingActive` – кормление выполняется сейчас
* `lastFeeding`, `nextFeeding` – последнее / следующее кормление
* `blocked`, `blockReason` – заблокировано сейчас + причина
* `lastResult`, `error` – результат последней попытки + флаг неисправности
* `feedNow` – ручной запуск (доступно для записи)

> Примечание: поиск адреса/геокодирование (Nominatim) и плитки карты требуют доступа в
> интернет. Геокодирование выполняется в бэкенде адаптера; экземпляр должен быть запущен.
