## Адаптер futterautomat для ioBroker

Керування (переобладнаними) автоматичними годівницями. Адаптер за розкладом вмикає та вимикає
до **5 довільно обраних перемикачів** (наявних об'єктів ioBroker) на налаштовуваний час
(«годування»). За потреби враховуються температура повітря та води, а за геокоординатами
обчислюється положення сонця, щоб не годувати вночі.

### Можливості

* До 5 перемикачів, кожен із довільною назвою (= окрема вкладка налаштувань).
* Два режими годування для кожного перемикача:
  * **Фіксований час** (напр., 08:00 і 18:00).
  * **Інтервал у межах часового вікна** (напр., кожні 60 хв між 08:00 і 18:00).
* Налаштовувана **тривалість годування в секундах** для кожного перемикача.
* **Блокування за температурою**: блокувати годування нижче або вище довільно заданих
  температур води та/або повітря.
* **Нічний захист** за сходом/заходом сонця з налаштовуваним зсувом (ранок/вечір).
* **Ручний запуск** (`feedNow`) для кожного перемикача та **кнопка «Погодувати зараз»** з
  обираною тривалістю; за бажанням з ігноруванням усіх блокувань.
* **Нагляд за перемиканням** для кожного перемикача: перевіряє, чи перемикач справді
  вмикається та вимикається (потрібен перемикач, що повідомляє свій стан, `ack=true`), з
  додатковими **сповіщеннями Telegram** (успішне годування / не виконано / несправність
  вимкнення).

### Налаштування

**Вкладка «Загальні налаштування»**
* **Місцезнаходження (обов'язково)**: використати системні налаштування *або* задати вручну –
  через пошук адреси та позначку на карті OpenStreetMap (ключ API не потрібен).
* **Сонячне вікно**: зсуви у хвилинах після сходу / до заходу сонця.
* **Джерела температури**: увімкнути температуру повітря та води й обрати відповідний об'єкт.
* **Перемикачі**: додати перемикачі (макс. 5), задати назви, обрати об'єкт, активувати.

**Вкладка перемикача** (створюється динамічно, з назвою перемикача)
* Режим (фіксований час / інтервал), час або вікно + інтервал, тривалість годування,
  блокування за температурою, нічні/ручні параметри, нагляд за перемиканням, сповіщення
  Telegram, кнопка ручного годування.

### Точки даних

Глобально:
* `info.connection` – адаптер працює / конфігурація дійсна
* `airTemperature`, `waterTemperature` – поточні виміряні температури
* `sunrise`, `sunset` – обчислений час

Для кожного перемикача в `switches.<id>.`:
* `feedingActive` – годування триває
* `lastFeeding`, `nextFeeding` – останнє / наступне годування
* `blocked`, `blockReason` – зараз заблоковано + причина
* `lastResult`, `error` – результат останньої спроби + прапорець несправності
* `feedNow` – ручний запуск (доступний для запису)

> Примітка: пошук адреси/геокодування (Nominatim) та плитки карти потребують доступу до
> Інтернету. Геокодування виконується в бекенді адаптера; екземпляр має бути запущений.
